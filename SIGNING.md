# Signing & verification

`noidle.app` releases are signed two ways:

1. **SHA256 checksums** — every release has a `SHA256SUMS.txt` listing
   the canonical hashes of `noidle.exe` and `noidle.msi`.
2. **cosign signatures** — every release has `.sig` and `.pem` files for
   each artifact, generated by [Sigstore cosign](https://docs.sigstore.dev/)
   keyless signing via GitHub OIDC. There is **no private key**; the
   identity is proven by a transparency-log entry tied to this repo's
   tag-push event.

There is **no Authenticode signature yet** (see [Status](#status) below).
That means Windows SmartScreen will still warn on first run — clicking
*More info → Run anyway* is the expected UX until Authenticode is in
place.

## Quick verify (any release)

```powershell
# 1. Download noidle.exe + SHA256SUMS.txt from the release page.
# 2. Verify the hash matches:
(Get-FileHash -Algorithm SHA256 noidle.exe).Hash.ToLower()
# Compare to the line for noidle.exe in SHA256SUMS.txt
```

```bash
# Or on macOS / WSL / Linux (verifying before transferring to a Windows box):
sha256sum -c SHA256SUMS.txt
```

## Cosign verify (cryptographic, optional)

If you have [cosign](https://github.com/sigstore/cosign) installed:

```bash
cosign verify-blob \
  --certificate noidle.exe.pem \
  --signature noidle.exe.sig \
  --certificate-identity-regexp "https://github.com/calebohara/noidle.app/.*" \
  --certificate-oidc-issuer "https://token.actions.githubusercontent.com" \
  noidle.exe
```

A successful verify confirms the binary was built by this repo's GitHub
Actions workflow on the tagged commit — same trust root as the public
GitHub Releases page, but cryptographically anchored in
[Rekor](https://rekor.sigstore.dev/) so it can't be silently swapped
later.

The `.msi` installer is signed the same way. Substitute `noidle.msi`,
`noidle.msi.pem`, and `noidle.msi.sig` in the command above.

## Status

| Layer | Status | Effect |
|------|--------|--------|
| SHA256 checksums | ✅ shipped from v0.3.6 | Tamper-evidence. A user who downloads from a mirror can confirm the bytes match. |
| Cosign signatures | ✅ shipped from v0.3.6 | Cryptographic proof of provenance. Sophisticated users + supply-chain scanners can verify. |
| Authenticode (Windows codesign) | ❌ not yet — needs a cert | Would eliminate SmartScreen warnings and improve corporate-AV scores. |

## Plan for Authenticode

The cheapest path is the [SignPath.io OSS Foundation](https://signpath.io/foundation):
free Authenticode signing for qualifying open-source projects.

**Application steps (manual, ~15 min):**

1. Visit https://signpath.io/foundation/apply
2. Submit the project URL: `https://github.com/calebohara/noidle.app`
3. Confirm: MIT licensed, public repo, no malware, single maintainer with
   verifiable identity (LinkedIn / GitHub history works for them)
4. Wait for approval (turnaround: ~1–2 weeks; they're a small team)
5. On approval, SignPath provisions a project + an API token

Once approved, this repo will add a SignPath integration step in
`build.yml` that pushes the unsigned `.exe` and `.msi` to SignPath, waits
for the signed artifacts, and substitutes them into the release. The
existing cosign signatures and SHA256SUMS continue to ship alongside.

**Alternatives** if SignPath OSS doesn't approve:

- **[Azure Trusted Signing](https://learn.microsoft.com/en-us/azure/trusted-signing/)** — Microsoft's pay-per-use signing service (~$10/month). Requires an Azure tenant and EV-style identity verification.
- **OV cert from SSL.com / Sectigo / DigiCert** — $100–300/year; private key on a hardware token shipped to the maintainer.

## Reporting a tampered release

If a `SHA256SUMS.txt` line doesn't match what you just downloaded, **do
not run the binary**. Open an issue on GitHub with the URL you
downloaded from and the hashes you observed.

For active exploitation, email `caleb.ohara@gmail.com` with "noidle" in
the subject line.
